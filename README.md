# OllamaDev

**Local AI coding agent powered by Ollama ŌĆö 100% private, zero dependencies.**

OllamaDev is a terminal coding agent that runs entirely against your local [Ollama](https://ollama.com) instance. Nothing leaves your machine ŌĆö no cloud, no API keys, no subscription. It's a single self-contained **vanilla PHP** binary (no Composer, no extensions beyond PHP core + curl), with diff-previewed edits, undo, an interruptible agent loop, real context management, and 60+ tools.

## Why OllamaDev

- **Local & private** ŌĆö every token is generated by *your* Ollama models. No data ever leaves the machine.
- **Free** ŌĆö no subscription; use any model you can run locally.
- **Vanilla PHP** ŌĆö one file, built by concatenation. No package manager, no runtime deps.
- **Safe by default** ŌĆö edits show a diff and ask before applying; every change is checkpointed and undoable; sub-agents are read-only unless you grant writes.

## Features

### Agent & editing
- **Streaming responses** with a real context window (`num_ctx`) so long tool output doesn't silently truncate history.
- **Diff preview before edits** ŌĆö `write`/`edit` show a colorized unified diff; confirm in `ask` mode, auto-apply (still shown) in `auto`.
- **Checkpoint & undo** ŌĆö every file change is snapshotted; `/undo` reverts the last one, `/checkpoints` lists them.
- **Interrupt** ŌĆö press **Ctrl-C** to abort an in-flight response or tool loop and return to the prompt (without quitting).
- **Regenerate** ŌĆö `/retry` re-runs the last turn for a fresh answer.
- **Native + fallback tool-calling** ŌĆö uses Ollama function-calling when the model supports it, with a robust text-format parser fallback for smaller models.
- **Sub-agents** ŌĆö the agent can delegate a focused sub-task with `task`; sub-agents are **read-only by default** and never more permissive than the parent.

### Context & input
- **`@file` mentions** ŌĆö `explain @src/foo.php` inlines the file into your message.
- **Image / vision input** ŌĆö attach images with `@image.png` or `/image <path>` for multimodal models.
- **Real token/context meter** ŌĆö `/status` and `/context` show actual Ollama token counts and context fill.
- **Auto-compaction** ŌĆö older messages are summarized by the model to stay within the window.
- **Markdown rendering** ŌĆö headings, lists, and code blocks are styled in the terminal.

### Models
- **Any Ollama model** ŌĆö llama, qwen2.5-coder, mistral, codestral, deepseek, gpt-oss, gemma, phi, and more.
- **Smart model switching** ŌĆö `/model qwen2.5-coder` resolves untagged names and offers to pull if missing.
- **Auto-pull** ŌĆö `/pull <model>` or `ollamadev pull <model>` streams a model download.

### Sessions
- **Persistent sessions** with resume ŌĆö `ollamadev resume` shows an interactive picker of recent sessions.
- **Project memory** ŌĆö `/init` (or `ollamadev init`) scans the project and generates an `OLLAMADEV.md` the agent reads on startup.

### Power tools
- **60+ built-in tools** ŌĆö files, search, shell, git, web fetch/search, code navigation.
- **Custom slash commands** ŌĆö drop prompt templates in `~/.ollamadev/commands/*.md` and invoke them as `/name`.
- **Hooks** ŌĆö run shell commands on `beforePrompt` / `afterEdit` via config.
- **MCP** ŌĆö connect Model Context Protocol servers (stdio + HTTP/SSE).
- **LSP** ŌĆö code navigation/diagnostics, plus an LSP server mode for editors.
- **Terminal multiplexer** ŌĆö named background terminal sessions.
- **Desktop GUI (ADE)** ŌĆö an optional Boson WebView app with a file browser and PTY terminal.
- **VS Code extension** ŌĆö inline completions and chat.

## Requirements

- **PHP 8.0+** (CLI) with curl ŌĆö that's it.
- **[Ollama](https://ollama.com)** running locally (`ollama serve`).
- At least one model pulled, e.g. `ollama pull qwen2.5-coder`.

> Recommended models for reliable tool use: **qwen2.5-coder**, **qwen3-coder**, **mistral**, **codestral**, **llama3.1**, **gpt-oss**.

## Installation

### From source (recommended)

```bash
git clone https://github.com/kennethyork/OllamaDev.git
cd OllamaDev

# Build + run the test suite + install to ~/.local/bin/ollamadev
make install

# or just build the single-file binary in place
./build.sh        # produces ./ollamadev
```

`make install` only installs if the build and the offline smoke suite pass, and it backs up the previously installed binary so you can `make rollback` instantly.

Make sure `~/.local/bin` is on your `PATH`, then:

```bash
ollamadev help
```

### Desktop app (ADE)

**OllamaDev ADE** (Agentic Development Environment) is an optional desktop GUI in `Desktop/ollamadev-ade/` ŌĆö a [Boson](https://github.com/boson-php/boson) WebView app built with vanilla PHP/JS/HTML/CSS (no front-end frameworks). It wraps the agent in a windowed interface with a file browser and a real PTY terminal.

> Unlike the CLI binary (which is dependency-free), the desktop app uses Composer for the Boson runtime and needs **PHP 8.4+**.

```bash
cd Desktop/ollamadev-ade
composer install

composer start    # dev: serve at http://localhost:8080
composer build    # compile a native desktop app (boson compile)
```

### VS Code extension

```bash
cd vscode-extension
code --install-extension ollamadev-extension.vsix
```

## Usage

### Interactive

```bash
ollamadev                      # start a chat session
ollamadev --auto               # auto-approve all tools (no prompts)
ollamadev --readonly           # plan mode: block all mutating tools
ollamadev -m qwen2.5-coder     # pick a model for this session
ollamadev resume               # pick a recent session to resume
ollamadev -c                   # continue the most recent session
```

### One-shot

```bash
ollamadev -p "explain @src/app.js"
echo "summarize this" | ollamadev chat
```

### Commands

```bash
ollamadev new            # new session
ollamadev list           # list sessions
ollamadev resume         # interactive resume picker
ollamadev load <id>      # load a session by id
ollamadev pull <model>   # download a model from Ollama
ollamadev init           # generate OLLAMADEV.md project memory
ollamadev models         # list installed models
ollamadev git <cmd>      # run git
ollamadev terminal ...   # terminal multiplexer
ollamadev lsp            # LSP server mode (for editors)
ollamadev help [topic]   # help (topics: commands, tools, git, terminal, session, tips)
```

### In-chat slash commands

Type `/help` in a session for the full, grouped list. Highlights:

| Command | Description |
|---------|-------------|
| `/model <name>` ┬Ę `/models` ┬Ę `/pull <m>` | switch / list / download models |
| `@path/to/file` | inline a file into your message |
| `/image <path>` | attach an image (vision models) |
| `/undo` ┬Ę `/checkpoints` | revert the last edit / list checkpoints |
| `/retry` | regenerate the last response |
| `/init` | generate `OLLAMADEV.md` project memory |
| `/context` ┬Ę `/status` | context fill & real token usage |
| `/permission <mode\|allow\|deny>` | manage tool approval |
| `/chat` ┬Ę `/agent` | toggle chat-only vs. tool-using mode |
| `/compact` ┬Ę `/new` ┬Ę `/tools` ┬Ę `/commands` | compact, new session, list tools/custom commands |
| `/cd` ┬Ę `/ls` ┬Ę `/pwd` ┬Ę `/git` ┬Ę `/save` ┬Ę `/clear` ┬Ę `/exit` | session & navigation |

The input line supports history (Ōåæ/Ōåō), cursor editing, **Tab completion** (commands, paths, model names), and **Ctrl-C** to interrupt a response.

### Permission modes

| Mode | Behavior | Flag |
|------|----------|------|
| `ask` (default) | prompts before mutating tools (shows the diff) | ŌĆö |
| `auto` | runs every tool without asking | `--auto` / `--yolo` |
| `readonly` | blocks all mutating tools (plan mode) | `--readonly` / `--plan` |

Switch at runtime with `/permission mode <auto\|ask\|readonly>`. Sub-agents run **read-only** regardless of the parent mode unless explicitly granted writes.

## Configuration

Config is read from the first of: `~/.ollamadev/config.json`, `~/.config/ollamadev/config.json`, `.ollamadev.json` (project-local).

```json
{
  "ollama": {
    "host": "http://localhost:11434",
    "defaultModel": "qwen2.5-coder:latest",
    "contextWindow": 16384,
    "temperature": 0.6,
    "stream": true
  },
  "agents": {
    "maxIterations": 12,
    "maxToolOutput": 12000,
    "subagentPermission": "readonly"
  },
  "permissions": { "mode": "ask" },
  "ui": { "markdown": true },
  "hooks": {
    "afterEdit": "echo edited"
  },
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/workspace"]
    }
  }
}
```

### Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `OLLAMA_HOST` | `http://localhost:11434` | Ollama API endpoint |
| `OLLAMA_MODEL` | first installed model | Default model |
| `OLLAMA_NUM_CTX` | `16384` | Context window size |

### Project memory

If `OLLAMADEV.md` (or `.ollamadev.md`) exists in the working directory, its contents are added to the system prompt. Generate one with `/init`.

### Custom commands & hooks

- Put a prompt template in `~/.ollamadev/commands/review.md` (supports `$ARGS` / `$1`) and invoke it as `/review ...`.
- Configure `hooks.beforePrompt` / `hooks.afterEdit` shell commands in config to run at those points.

## Data storage

- Sessions: `~/.ollamadev/sessions/`
- Checkpoints: `~/.ollamadev/checkpoints/`
- Terminals: `~/.ollamadev/terminals/`
- Custom commands: `~/.ollamadev/commands/`

## Development

The binary is built by concatenating `src/[0-9]*.php` (only `src/00-header.php` carries the `<?php` tag). **Edit the modules in `src/`, never the built binary.**

```bash
./build.sh                 # build ./ollamadev (php -l validated)
php tests/smoke.php         # offline, deterministic tests (no model needed)
SMOKE_MODEL=qwen3-coder:latest php tests/smoke.php   # + end-to-end tests against a real model
make                        # build + test
make install                # build + test + install (with backup)
make rollback               # restore the previously installed binary
scripts/safe-apply.sh "msg" # build ŌåÆ test ŌåÆ install ŌåÆ commit, auto-reverting on failure
```

## Troubleshooting

**Cannot connect to Ollama** ŌĆö start it with `ollama serve`, or set `OLLAMA_HOST`.

**Model not found** ŌĆö `ollama pull qwen2.5-coder`, or just `/pull qwen2.5-coder` in-session.

**Unreliable tool use** ŌĆö small models struggle with tool-calling; try a recommended model (qwen2.5-coder, mistral, codestral) or use `/chat` for plain conversation.

**Permission denied running the binary** ŌĆö `chmod +x ~/.local/bin/ollamadev`.
