The canvas

Everything is a window. The canvas is infinite.

No tabs, no fixed panels. Terminals, the editor, file tree, code search, task board, memory graph, a localhost browser preview, voice control, and the crew topology are all draggable, resizable windows you pan and zoom across.

  • Drop a new window from ＋ Add or by right-clicking the canvas
  • Real PTY terminals with full-screen TUI support (vim, htop)
  • Each project remembers its own canvas — windows, positions, session
  • The rail + Projects switcher tuck into a ☰ drawer so the canvas is full-bleed
The crew cockpit

Watch a whole team of agents build — locally.

A Director plans and assigns roles; Coders each work in an isolated git worktree; an Auditor reviews every diff and sends flagged work back for one fix before it lands. The live topology shows it all as it happens.

  • Each coder's branch, the files it's touching, and the Auditor's verdict
  • Real-time activity — ✎ editing · 👁 reading · ⚡ running
  • Launch & steer it by voice: "start a crew to add tests"
  • Run coders in parallel on one box; pick a model per role

The private, free answer to cloud agent-canvases like cnvs, BridgeSpace, and Plyrium Forge — but it runs entirely on your hardware.

Skills

60+ built-in skills the crew loads on demand.

Reusable SKILL.md instructions the agent loads via the skill tool. Capability skills (payments-money, testing-discipline, auth-security…) plus a per-team starter for every crew team — pick the E-commerce team and it auto-loads the ecommerce skill.

  • Full CRUD in the Skills window — create, view, edit, delete
  • Customize any built-in; your copy overrides it for every run
  • Shared across CLI, desktop, and web; discover more with skills browse
Features

Everything a coding agent needs — locally

60+ built-in tools, real context management, and an interruptible agent loop. No telemetry, ever.

🔒

Local & private

Every token is generated by your own models on your hardware. No data ever leaves the machine.

💸

Free forever

No subscription, no per-token billing. Run any model your GPU (or CPU) can handle.

📦

Zero dependencies

One vanilla-PHP file built by concatenation. No Composer, no npm, no framework — auditable in one read.

🛡️

Safe by default

Edits show a colorized diff and ask first. Every change is checkpointed and reversible with /undo — and ollamadev verify runs your tests and fixes failures until they pass.

👥

Crew — multi-agent

Director plans, coders build in isolated git worktrees, an Auditor gates every diff. Per-role models, an adversarial audit panel, and a live topology you watch.

🕸️

Graph memory

A wiki-linked knowledge base the agent recalls on demand — [[linked]] notes that survive across sessions, rendered as a graph.

🎤

Voice control

Press-to-talk with a bundled, on-device Whisper engine. Dictate to the CLI, open windows, or launch and steer the crew — 100% offline.

🔌

Code intelligence

MCP servers (stdio + HTTP/SSE), real LSP navigation & diagnostics, and local semantic code search embedded with your own model. No cloud.

🛜

Air-gapped & attestable

Flip on --offline and every network tool is hard-blocked — unwaivable. ollamadev attest prints a fingerprint proving the run never touched the wire.

🌐

CLI · desktop · web

The terminal, a native desktop app, or your browser (composer serve) — one engine, one set of data. Crew runs, memory, models, and skills are shared across all three.

🧩

Skills

60+ built-in — capability skills (payments-money, testing-discipline…) plus a per-team starter — that crews load by focus. View, create, edit, or delete any in the Skills window; customize a built-in to override it.

♻️

Picks up where you left off

Close the app or reboot — sessions resume per repo, a running crew comes back from disk, and each project restores its own canvas: windows, positions, and session.

Install

Up and running in two commands

Requires PHP 8.0+ with curl, and Ollama running locally.

1 Start Ollama

# install a model
ollama pull qwen2.5-coder
ollama serve

2 Install OllamaDev

# PHP-free native binary
curl -fsSL https://raw.github\
usercontent.com/kennethyork/\
OllamaDev/main/install.sh | sh

3 Code

# terminal, or open the ADE
ollamadev
ollamadev crew "add tests"

Prefer a GUI? Grab the desktop app (ADE) or run browser/web mode (composer serve) — same canvas, same local engine. There's a VS Code extension too — see CLI · Desktop · Web and the self-hosting docs.
