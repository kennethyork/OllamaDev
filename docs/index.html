Features

Everything a coding agent needs — locally

60+ built-in tools, real context management, and an interruptible agent loop. No telemetry, ever.

🔒

Local & private

Every token is generated by your own models on your hardware. No data ever leaves the machine.

💸

Free forever

No subscription, no per-token billing. Run any model your GPU (or CPU) can handle.

📦

Zero dependencies

One vanilla-PHP file built by concatenation. No Composer, no npm, no framework — auditable in one read.

🛡️

Safe by default

Edits show a colorized diff and ask first. Every change is checkpointed and reversible with /undo — and ollamadev verify runs your tests and fixes failures until they pass, so changes actually work.

👥

Crew — multi-agent

A Director plans and assigns each subtask a role (coder, tester, docs, security, or your own), parallel agents build in isolated git worktrees, an Auditor reviews. Per-role models, multi-host.

🧩

Skills

Reusable SKILL.md packs with progressive disclosure. Install, export, and share them.

🕸️

Graph memory

A wiki-linked knowledge base the agent recalls on demand — context that survives across sessions.

🔌

Code intelligence

Connect MCP servers (stdio + HTTP/SSE) and get real LSP navigation & diagnostics — plus local semantic code search: ollamadev index build embeds the repo with your own model, so the agent finds code by meaning, not keywords. No cloud.

🖥️

Two local backends

Ollama by default; opt into LM Studio for a higher model ceiling. Both stay on localhost.

🌐

CLI · desktop · web

Run it in the terminal, as a native desktop app, or in your browser (composer serve) — same UI, same local engine, shared data. Use it from your phone via a tunnel.

🛜

Air-gapped & attestable

Flip on --offline and every network tool is hard-blocked — unwaivable. ollamadev attest prints a fingerprint proving the run never touched the wire. Built for regulated and disconnected work.

♻️

Picks up where you left off

Close the app or reboot — sessions resume per-repo and a running Crew comes back from disk, right where it stopped. Each folder is its own persistent workspace.

🎤

Voice input

Dictate to the agent with a local speech-to-text engine of your choice. Engine-agnostic (stt.host or stt.command) and, like everything else, it stays on your machine.

🗂️

Workspaces

A Projects list in the left pane of the app (or ws in the terminal). Click a project to switch — each restores its own terminals, editor tabs, layout, and session. One global list shared by CLI, desktop, and web.

🛰️

Yours, from anywhere

Self-host on a VPS and drive it from your phone over an SSH tunnel or Tailscale. Localhost-only by default with a required token, so it's reachable by you alone — never the open internet.

Install

Up and running in two commands

Requires PHP 8.0+ with curl, and Ollama running locally.

1 Start Ollama

# install a model
ollama pull qwen2.5-coder
ollama serve

2 Build & install

git clone https://github.com/\
  kennethyork/OllamaDev
cd OllamaDev
make install

3 Code

# interactive chat
ollamadev

# or run a crew
ollamadev crew "add tests"

Prefer a GUI? There's a desktop app (ADE), a browser/web mode (run it on a server and use it anywhere), and a VS Code extension too — see CLI · Desktop · Web and the self-hosting docs.
